Kilotest

Kilotest performs a thousand tests on a web page.

A thousand? Really?

Ensemble testing

Kilotest does ensemble testing, with a focus on accessibility.

Software fails unless tested. Efficient testing first leverages automated tools and corrects the defects that they discover, then employs human testers.

Numerous automated testing tools exist. They vary in what and how they test. Tools are partly complementary (they discover different issues) and partly duplicative (they test for the same issues).

Ensemble testing takes advantage of both these facts. The complementarity allows testing for a larger set of issues. The duplicativity allows tools to act as checks on one another’s quality.

The tools

The Kilotest ensemble is 11 tools developed by teams in the USA, Canada, Denmark, Australia, and Portugal.

The tools perform more than a thousand tests in total. Open-source software developed initially at CVS Health drives the tools, tries to weed out invalid results, and distills the findings into about 300 issues.

Status

Kilotest is a proof of concept, launched in November 2025. Your suggestions, bug reports, and collaboration proposals are welcome! Post them on GitHub or email them to info@kilotest.com.

In the current version, users can get results of already performed tests and can recommend new tests. Kilotest managers can approve recommendations.
